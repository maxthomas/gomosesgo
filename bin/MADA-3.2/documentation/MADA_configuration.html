MADA+TOKAN is controlled by means of several configuration variables that can be defined in a configuration file, or provided on the command line. When MADA is installed, it creates a default configuration file in the MADA home directory, under "config-files/template.madaconfig". This file lists every configuration variable that MADA and TOKAN use, and provides documentation for each. This documentation is also presented below, divided into categories.

MADA configuration files are formatted so that only one variable (at most) is defined on each line, using a

    Variable Name = Variable Value

format (see the default template.madaconfig file for examples). Blank lines are ignored, and anything appearing after a "#" character is ignored as a comment. By convention, variable names are all upper-case; for the most part, variable values are case-insensitive.

Mandatory Directory Settings

The following settings are used to specify the location of the MADA home directory and the third-party tools MADA uses. If the MADA INSTALL.pl installation script was run correctly, the template.madaconfig file it created should already have these variables set properly. Otherwise, users will have to manually set them to the proper locations on their systems before running MADA.

Mandatory VariablePossible Values Default ValueDescription
MADA_HOMEAny valid directory Defined by INSTALL.plThe full-path directory location of the MADA installation
SRI_NGRAM_TOOLThe "disambig" executable file Defined by INSTALL.plThe full-path location of the SRILM "disambig" tool
SVM_TAGGERThe "SVMTagger" executable file Defined by INSTALL.plThe full-path location of the SVMTool "SVMTagger" tool

General Use Settings

These settings are used for general control of MADA and TOKAN; most users will only ever use these variables to adjust MADA and TOKAN to their needs.

Control Variables

Control VariablePossible Values Default ValueDescription
RUN_PREPROCESSORYES | NO YESShould the MADA Preprocessor be run?
RUN_MADAYES | NO YESShould MADA be run?
RUN_TOKANYES | NO NOShould TOKAN be run?

Input Format Variables

Input VariablePossible Values Default ValueDescription
INPUT_ENCODINGBuckwalter | BW | UTF8 | UTF-8 UTF8Is the input file encoded as Buckwalter (BW) or UTF-8 (UTF8)? If UTF-8, the MADA Preprocessor will need to be run to convert the data, as MADA needs to use Buckwalter encoding internally.
SEPARATEPUNCTYES | NO YESIf YES, MADA try to separate words from surrounding punctuation and digits, a necessary step for good performance. This will also break up sequences of punctuation and common number formats, but MADA will avoid breaking numbers with decimal points. If NO, users are encouraged to preprocess their input files to separate digits/puncutation.
SENTENCE_IDSYES | NO NOIf YES, MADA+TOKAN will treat the first word of every input line as a sentence ID token, which will not be processed and will be carried foward throughout the pipeline. The sentence IDs can be any sequence of non-whitespace characters, but if used, one must be present on every line of the input.

Output Format Variables

Output VariablePossible Values Default ValueDescription
SORT_COMBINER_OUTPUTYES | NO YESIf YES, the final MADA analysis lists will be sorted according to their ranking score. If NO, the lists will be in the order produced by the morphological analyzer.
PRINT_ANALYSESall | stars | [number] allIf all, the MADA output will list all the generated analyses for each word. If stars, the output will list only the top-ranked analysis for each word (significantly reducing the output file size). If [number], the MADA output will list the analyses that have the [number] top-scores for each word.

Combiner / Selection Variables

Combiner / Selection VariablePossible Values Default ValueDescription
FEATURE_WEIGHT_SETnormal | original | pos | lexeme | custom normalWhich pre-calcuated feature weight set should be used? The normal weightset is tuned for the best overall analysis choice. The original weightset equally weights all SVM features. The pos and lexeme weightsets are tuned to provide the best choices for part-of-speech and lemma selection, respectively. Finally, it is possible for users to define their own feature weightsets by selecting custom and then pointing to the feature weight file with CUSTOM_FEATURE_WEIGHT_FILE (below).
CUSTOM_FEATURE_WEIGHT_FILE A file containing user-defined feature weights Same as normalIf FEATURE_WEIGHT_SET is set to custom, this variable needs to be set to the user-defined feature weightset file. The format of this file should be consistent with the files found in "MADA_HOME/feature-weights"
TIE_BREAKINGarbitrary | random | none arbitraryWhen two or more analyses have tie scores, what should be done? If arbitrary, the top analysis in the list will be selected, and the others will be marked with a "^". If random, one of the tied analyses will be randomly selected, and the others will be marked with "^". If none, all tied analyses will be selected (marked with "*" in the output file).

TOKAN Variables

TOKAN VariablePossible Values Default ValueDescription
TOKAN_SCHEME A highly customizable option string NAThis variable allows highly customized tokenizations to be defined. A full description of possible options can be found in the MADA+TOKAN manual.
TOKAN_OUTPUT_EXTENSIONa string suitble for filenames tokThis string will be used as a filename extension of the TOKAN output file
TOKAN_SCHEMES_FILEA file containing a set of TOKAN schems NAIf set to a properly formated file (see "MADA_HOME/config_files/TOKAN.schemes" for an example), TOKAN will produce an output file for every scheme listed in this file, each using an output file extension specified in the schemes file.

Cleanup Variables

Cleanup VariablePossible Values Default ValueDescription
REMOVE_TEMP_NGRAM_FILESYES | NO YESIf YES, the temporary diacritic and lemma N-gram files MADA generates will be automatically deleted.
REMOVE_BACKLEX_FILESYES | NO YESIf YES, MADA will automatically delete the backup lexicon files it generates after SVMTools is finished with them.
RUN_MA_FILEYES | NO NOIf YES, MADA will automatically delete the initial morphological analyzer file, the SVMTools output files, and the N-gram summary file it generated once the final output file is generated.
COMPRESS_OUTPUTSYES | NO NOIf YES, MADA will automatically compress (gzip) any output files it has produced (including temporary files if they were not automatically deleted).

Advanced Settings

In addition to the above variables, MADA configuration files contain a few additional settings that control the exact models and features used by MADA. Since currently MADA is only released with a single set of models, most users will never have a need to adjust these variables. It is recommended that users do not alter any of these variables from their default values.

Advanced VariablePossible Values Default ValueDescription
ALMOR_DATABASEAn ALMOR database file almor.dbThe ALMOR database file to be used for morphological analysis. It is assumed to be located under "MADA_HOME/MADA/." Users should never have to alter this.
MORPH_BACKOFFnone | noan-all | noan-prop | add-all | add-prop noneThis variable controls whether the morphological analyzer will create additional "backoff" analyses that may be useful in identifying rare words, but are not as accurate as regular analyses. See the default "generic.madaconfig" file for more details.
NGRAM_LM_DIRECTORYA subdirectory under "MADA_HOME" models/This is the subdirectory containing the N-gram models used in MADA. Users should never have to change this.
LEXEME_TYPEBAMA | NORM NORMThis controls whether the lexemes modeled in MADA utilize the numeric underscore suffixes used in the BAMA/SAMA database files (BAMA) or drops them (NORM). Users should never have to change this.
LEX_NGRAM_ORDER1 | 2 | 3 | 4 | 5 4This is the order of the lexeme N-gram models used by MADA. Tests have shown 4-grams to have a good balance between accuracy and computation time, so that is the default. Users should never have to change this.
DIAC_NGRAM_ORDER1 | 2 | 3 | 4 | 5 5This is the order of the diacritic form N-gram models used by MADA. Tests have shown 5-grams to have a good balance between accuracy and computation time, so that is the default. Users should never have to change this.
CLASSIFIERSA subset of 14 specific classifier names All 14 classifiersThis is the list of classifiers for which MADA will run SVMs. Only these classifiers will be used to make predictions of the correct analysis. Note that if a given classifier is excluded, MADA will still generate a prediction for it since that information comes from the chosen analysis. Note also that if this variable is changed from the default, the feature weights used will no longer be optimized, which will likely result in poorer performance (but faster processing speed, since fewer SVMs will be run).
OTHER_FEATURESA subset of the 10 non-SVM features All 10 featuresThis is the list of non-SVM features that MADA will use in determining the best analysis. Note also that if this variable is changed from the default, the feature weights used will no longer be optimized. Users are recommended to use the default (full) list.
MODEL_DIRA subdirectory under "MADA_HOME" models/This is the subdirectory containing the SVMTools model files used by MADA. Users should never have to change this.
